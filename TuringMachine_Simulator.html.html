Turing Machines — Theory

A Turing Machine (TM) is the most powerful computational model in the Chomsky hierarchy. Proposed by Alan Turing in 1936, it forms the theoretical foundation of modern computing and defines the boundary between what is and is not computable.

Formal Definition

A Turing Machine is a 7-tuple:

M = (Q, Σ, Γ, δ, q₀, q_accept, q_reject)

Q — finite set of states
Σ — input alphabet (not containing blank symbol □)
Γ — tape alphabet (Σ ⊆ Γ, □ ∈ Γ)
δ — transition function: Q × Γ → Q × Γ × {L, R}
q₀ ∈ Q — start state
q_accept — accept state
q_reject — reject state (q_accept ≠ q_reject)

How It Works

The TM operates on an infinite tape divided into cells. At each step, it reads the current cell under the head, consults the transition function δ, then writes a symbol, moves left or right, and transitions to a new state. Computation halts when it enters q_accept or q_reject.

Key Concepts Demonstrated in this Simulator

Configurations

A configuration (q, tape, head) captures the complete instantaneous description of a TM. The simulator displays each configuration after every step.

Decidability

All 4 machines here are deciders — they always halt on every input. The aⁿbⁿ language is undecidable by DFAs but decidable by a TM.

Computation

TMs aren't just language recognizers — the unary adder proves they can compute functions, underpinning the Church-Turing thesis.

Tape as Memory

Unlike finite automata, the TM's infinite read-write tape provides unbounded memory, enabling recognition of non-regular and non-context-free languages.

The Halting Problem

Turing proved that no TM can decide whether an arbitrary TM halts on a given input — this is the Halting Problem, the first undecidable problem. It establishes the fundamental limits of computation and is proven by diagonalization. All 4 machines in this simulator always halt, making them decidable instances.

Church-Turing Thesis

Any effectively computable function can be computed by a Turing Machine. This means TMs capture the full power of computation — no physically realizable model is more powerful (though some may be faster).

Why Turing Machines?

Turing Machines are the most fundamental concept in all of Theory of Computation. They define the limits of what computers can and cannot do, form the basis for complexity theory (P vs NP), undecidability, and the Church-Turing thesis. Simulating a TM requires implementing all core elements: states, tape alphabet, transition functions, halting conditions, and formal configuration semantics.
